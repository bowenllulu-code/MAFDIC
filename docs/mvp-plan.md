# MVP Plan

## 1. MVP 目标

阶段 1 的目标是把 MAFDIC 从产品方案推进到可建设的 MVP 范围。

第一版 MVP 不追求真实后端、不追求完整流程引擎，也不追求生产级 AI 执行能力。它要先完成三件事：

- 建立一个专业、可信、可演示的基金代销运营控制台
- 用 mock 数据跑通核心业务视图和关键用户路径
- 用 typed adapter 边界为后续真实业务 API 接入做好准备

## 2. MVP 用户

### 专职运营人员

使用系统完成日常查询、核对、异常处理、配置查看、报表准备和任务跟踪。

### 高层管理者

使用系统查看经营分析、风险异常、客户贡献、商机转化和业绩归因。

## 3. MVP 页面范围

### 3.1 综合工作台

目标：作为运营人员和管理者的统一入口。

第一版内容：

- 今日关键指标
- 待办任务
- 风险异常摘要
- 交易确认异常摘要
- 经营指标摘要
- 近期报表和定时任务
- AI 建议入口

### 3.2 客户全景图

目标：围绕一个机构客户展示客户、账户、交易、持仓、收益、商机和风险。

第一版内容：

- 客户基本信息
- 联系人和账户信息
- 交易摘要
- 持仓摘要
- 收益摘要
- 商机关联摘要
- 风险和异常摘要
- 客户相关 AI 问答入口

### 3.3 交易与资产中心

目标：支持运营人员查询交易、持仓、资产和收益变化。

第一版内容：

- 交易记录列表
- 订单状态轨迹
- 持仓明细
- 基金行情和净值摘要
- 资产变化归因
- 收益、分红、费用摘要

### 3.4 确认与清算视图

目标：解释交易申请、TA 回执、确认结果和清算对账之间的关系。

第一版内容：

- 确认状态列表
- 申请与确认映射
- 回执明细
- 清算记录
- 对账差异摘要
- 未确认原因 mock 解释

### 3.5 风险异常看板

目标：展示风险事件、告警、处置状态和 AI 解释。

第一版内容：

- 风险事件列表
- 告警等级和状态
- 命中规则
- 原因解释
- 处置建议
- 闭环结果

### 3.6 经营分析看板

目标：为管理者提供资产、交易、收入、费用和商机转化视图。

第一版内容：

- 资产规模趋势
- 交易量趋势
- 收入和费用摘要
- 客户贡献排行
- 产品贡献排行
- 渠道贡献排行
- 商机转化摘要

### 3.7 运营配置中心

目标：展示垫资、垫资行、孳息和费用规则的配置工作台雏形。

第一版内容：

- 投资者垫资配置列表
- 垫资行列表
- 孳息规则列表
- 费用规则占位
- 配置状态
- 版本和审批状态
- 配置变更摘要

第一版只做查看、筛选、详情和 mock 校验，不做真实配置提交。

### 3.8 商机与业绩归因中心

目标：展示商机从客户、交易到收入和业绩的可追溯链路。

第一版内容：

- 商机列表
- 商机阶段
- 关联客户
- 关联交易
- 收入和费用贡献
- 业绩归因结果
- 归因链路说明

### 3.9 AI 助手工作台

目标：验证 AI 嵌入运营流程的交互方式。

第一版内容：

- 对话区
- 推荐问题
- 业务上下文卡片
- 报表生成预览
- 定时任务创建预览
- 邮件草稿预览
- 配置影响分析预览
- 人审确认提示

第一版使用 mock AI 响应，不接真实模型。

## 4. MVP 关键用户路径

### 路径一：客户到交易资产

1. 用户进入客户全景图
2. 搜索或选择机构客户
3. 查看账户状态、交易摘要、持仓和收益
4. 下钻到交易与资产中心
5. 查看某笔交易的状态轨迹和资产影响
6. 向 AI 助手询问“该客户资产为什么变化”

### 路径二：商机到业绩归因

1. 用户进入商机与业绩归因中心
2. 查看商机阶段和关联客户
3. 下钻关联交易
4. 查看交易带来的收入、费用和业绩
5. 生成商机转化说明

### 路径三：风险异常到处置建议

1. 用户进入风险异常看板
2. 查看高优先级告警
3. 打开告警详情
4. 查看命中规则、原因解释和相关交易
5. 获取 AI 处置建议
6. 生成待办或处理说明

### 路径四：运营配置到审批说明

1. 用户进入运营配置中心
2. 查看垫资或孳息配置
3. 打开配置详情和版本历史
4. 查看 mock 校验结果
5. 让 AI 生成配置变更影响说明
6. 预览审批说明

### 路径五：经营分析到定时报表

1. 管理者进入经营分析看板
2. 查看收入、交易、资产和商机转化指标
3. 选择报表模板
4. 让 AI 助手生成周报预览
5. 创建 mock 定时推送任务

## 5. MVP 暂不做

- 真实业务 API 接入
- 完整后端服务
- 真实登录和权限体系
- 真实审批流
- 真实邮件发送
- 真实定时任务执行
- 真实 AI 模型调用
- 复杂可视化大屏
- 移动端适配

## 6. MVP 验收标准

- 可以本地启动前端应用
- 一级导航和核心页面完整
- 页面之间可以通过关键对象互相跳转
- mock 数据覆盖客户、账户、交易、持仓、行情、商机、营收、配置、风险、任务
- adapter 接口和 domain model 明确
- AI 助手有 5 个以上 mock 业务动作预览
- 页面风格符合专业运营控制台，不是营销站点
