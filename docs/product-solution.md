# MAFDIC Product Solution

## 1. 产品定位

MAFDIC（Multi-Agent Fund Distribution Insight Claw）定位为面向机构投资者基金代销运营场景的智能运营中台。

系统不只是数据看板，也不只是业务配置后台，而是将客户、账户、交易、资产、清算、风险、商机、业绩、运营配置、审批、知识质检和 AI 助手整合到一个统一工作台中，帮助运营人员完成日常作业，帮助管理者获得经营洞察，并通过多 Agent 能力自动化报表、任务、解释、预警和触达。

## 2. 目标用户

### 2.1 专职运营人员

负责日常数据查询、交易跟踪、确认清算核对、垫资与孳息配置、异常处理、报表输出和业务协同。

核心诉求：

- 快速定位客户、账户、交易、持仓、收益和异常
- 安全维护垫资行、垫资规则、孳息规则等运营配置
- 复用 SOP、审批、质检和知识库，减少人工判断偏差
- 通过 AI 辅助完成解释、汇总、待办生成和报表输出

### 2.2 高层管理者

关注业务整体经营、客户贡献、营收趋势、商机转化、风险变化和运营效率。

核心诉求：

- 快速查看经营分析和风险异常
- 理解收入、资产、交易和商机之间的关系
- 看到可解释、可追溯、可下钻的管理指标
- 接收定时报告、异常提醒和关键事项摘要

## 3. 设计原则

- **运营可信**：配置、审批、AI 动作、报表和数据口径都必须可审计、可追溯。
- **数据可解释**：交易、资产、收益、营收、商机和业绩归因不能只给结果，要能解释来源和变化原因。
- **AI 嵌入业务**：AI 不是孤立聊天框，而是嵌入查询、配置、审批、异常、报表和任务流。
- **前期 mock，后期适配**：外部交易、行情、客户、商机等 API 后续提供，当前先用 mock 数据和 adapter 边界建设。
- **配置优先安全**：垫资、孳息、费用、规则类配置必须支持版本、审批、生效期、回滚和留痕。

## 5. 核心业务中心

### 客户与账户中心

核心对象：客户、联系人、交易账户、基金账户、客户身份、账户状态。

主要输出：客户 360 视图、账户状态、客户交易摘要、持仓摘要、收益摘要、商机摘要，以及面向 AI 的客户与账户上下文。

### 交易与资产中心

核心对象：交易申请单、交易订单、持仓、份额、资产、收益、分红、费用、基金行情与净值。

主要输出：订单详情、订单状态轨迹、持仓资产明细、资产变化归因、收益与费用结果、基金行情与净值变动。

### 交易确认与清算中心

核心对象：TA 回执、确认单、确认金额、确认份额、清算记录、对账差异。

主要输出：确认状态、确认结果、回执明细、申请与确认映射关系、清算与对账差异分析。

### 风险与异常中心

核心对象：风控规则、风险事件、告警、告警处置记录、风险说明。

主要输出：告警列表、告警详情、命中规则、原因解释、处置建议、闭环结果。

### 运营配置中心

核心对象：投资者垫资配置、垫资行、孳息规则、费用规则、配置版本、生效期、配置审批、配置审计。

主要输出：垫资规则、垫资行维护结果、孳息配置、费用配置、配置变更历史、生效与失效状态、配置风险提示。

### 商机与业绩归因中心

核心对象：销售商机、商机关联客户、商机关联交易、商机关联基金、收入记录、费用记录、业绩归因规则、业绩结果。

主要输出：商机生命周期、商机转化结果、关联交易明细、收入与费用贡献、客户/渠道/产品/销售维度业绩、商机到业绩的可追溯链路。

### 作业与案件中心

核心对象：任务、工单、案件、处理意见、协同记录。

主要输出：待办任务、任务流转轨迹、案件全貌、处理结果、自动建单和补全上下文建议。

### 审批与授权中心

核心对象：审批模板、审批实例、审批节点、审批意见、授权记录。

主要输出：审批状态、待审列表、审批轨迹、授权结果、AI 动作的人审边界。

### 知识与质检中心

核心对象：SOP、规则说明、FAQ、案例库、制度文档、质检结果、整改记录。

主要输出：知识内容、处理指引、案例参考、质检结果、话术素材、AI 检索增强上下文。

## 6. AI 助手与多 Agent 方案

MAFDIC 的 AI 助手应从“对话框”升级为“智能助手工作台”。它既能对话，也能调用业务能力、生成任务、组织 Agent、输出报表并推动协同。

### 6.1 助手核心能力

- 业务问答
- 数据分析
- 报表生成
- 定时任务
- 邮件草稿与推送
- SOP 推荐
- 配置影响分析
- 异常解释

### 6.2 候选 Agent

- 数据分析 Agent
- 报表 Agent
- 调度 Agent
- 邮件 Agent
- 配置 Agent
- 风险 Agent
- 知识 Agent

### 6.3 AI 动作治理

AI 可以建议、草拟、解释和编排，但涉及以下动作时必须有人审确认：

- 修改业务配置
- 创建或变更审批
- 发送外部邮件
- 创建周期任务
- 导出敏感数据
- 标记风险事件闭环
- 触发批量操作

## 7. MVP 建设边界

第一阶段目标是做出可运行、可演示、可迭代的运营控制台。

### 7.1 第一版应包含

- 专业运营控制台框架
- 综合工作台
- 客户全景图
- 交易与资产查询
- 经营分析看板
- 风险异常看板
- 运营配置中心
- 商机与业绩归因视图
- AI 助手工作台
- mock 数据与 typed adapter 边界

### 7.2 第一版暂不包含

- 真实业务 API 对接

- 真实邮件发送
- 真实定时任务执行
- 生产级权限系统

这些能力需要保留架构边界，等业务 API、权限体系和生产集成要求明确后逐步接入。

## 8. 外部 API 适配策略

业务侧后续会提供交易记录、基金行情、客户信息、商机信息等 API。MAFDIC 前期不重复建设这些源系统能力，而是通过 adapter 适配。

### 8.1 适配对象

- 客户信息 API
- 账户信息 API
- 交易记录 API
- 交易确认与清算 API
- 基金行情与净值 API
- 持仓与资产 API
- 营收与费用 API
- 商机信息 API
- 风险事件 API

### 8.2 Adapter 职责

- 将外部字段转换为 MAFDIC 领域模型
- 屏蔽外部分页、过滤、错误码和状态码差异
- 保留原始业务 ID，确保可追溯
- 为 UI、报表和 Agent 提供稳定查询接口
- 支持后续缓存、权限过滤、审计和数据血缘
